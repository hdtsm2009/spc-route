🔧 仕組みと店舗選定基準

このツールが「どんな店を、どういう基準で、どの順番で回るか」をどう決めているかの解説です。

全体の流れ（2ステップ）

① 起点・時間帯を指定→ 候補を検索・スコア順に提示→ ② 行く店をチェック→ 巡回順を最適化→ Googleマップ用ルート発行

「どの店に行くか（選定）」と「どう回るか（順番）」を分けているのがポイントです。 担当者がスコアを見ながら自分で行く店を取捨選択でき、選んだ後の並べ替えは機械が最短に近い順で行います。

起点は エリアチップ・駅名/住所の自由入力・店名 のいずれでも指定でき、 「すでに行く予定のアポ地点」を起点周辺の中心に据えることもできます。

① どの店を「候補」として出すか

起点の座標から半径内（既定 約1,200m、変更可）にある店舗のうち、次の条件をすべて満たすものだけを候補にします。

  • 営業対象から除外されていない（過去のNG・取引不可フラグ、スペース貸し業態などは除外）
  • 座標があり、ジオコーディング品質が一定以上（地図がずれる低品質データは除外）
  • 指定時間帯に営業している可能性がある（営業時間情報から判定。定休・閉店明記は除外、不明は「要確認」として残す）
💡 スポカフェ掲載済みの店も候補に含めます。掲載店の多く（実データで約96%）は無料掲載で、 有料プランへの転換が見込めるためです。「掲載済み＝対象外」ではなく「掲載済み＝集客意欲のある営業先」と捉えます。 営業時間が不明な店も切り捨てず「🟡 要確認」として残します（現場で電話確認する前提）。
📍 スポカフェ掲載店の地名概算補完。スポカフェ掲載店マスタには番地が無く、 市区町村＋地名タグ（例「新宿三丁目」）しかありません。スクレイプ側に見つからない掲載店のうち、 地名が町名レベルまで特定できたものだけを「概算位置(町名)」として候補に加えています （町名の中心点であり店の正確な位置ではないため、現地で要確認）。町名まで絞れない店は誤誘導を避けるため候補に含めません。

② どの店を「おすすめ」するか（ランクとスコア）

ランクは「スポカフェ／ファンスタへの掲載状況」で決まります。 掲載している店ほどスポーツ集客への意欲が高く、有料プランへの転換や競合からの奪取が見込めるため上位にします。 候補は同ランク内で「店の魅力・温度（業態・評価・観戦設備など）」のスコア順、同点なら起点に近い順に並びます。

ランク掲載状況営業の狙い
Sスポカフェ＋ファンスタ両方掲載最優先。意欲が最も高く有料転換しやすい
ASスポカフェのみ掲載高優先。無料→有料転換／上位プランへのアップセル
AFファンスタのみ掲載高優先。競合のみ掲載＝スポカフェへの奪取
B両方未掲載（有望）新規開拓。スポーツバー・観戦設備あり等
C両方未掲載（温度低め）低優先。業態・ニーズ要確認
💡 スポカフェ掲載店の多く（実データで約96%）は無料掲載です。 無料掲載店は「すでにスポカフェを使う意思があるのに課金していない」＝有料転換の本命。 有料プラン契約済みの店も、上位プランへのアップセル対象として候補に残します。

各店には「なぜおすすめか」を示すひとことピッチが付きます。掲載状況に応じて次のように出ます。

  1. 両サービス掲載◎ … スポカフェ・ファンスタ両方に掲載。最有力（集客意欲が最も高い）
  2. スポカフェ無料→有料転換 … 無料掲載中。有料プランへの転換を狙う本命
  3. スポカフェ◯◯→上位 … 有料プラン契約中。上位プランへのアップセル
  4. ファンスタのみ→奪取 … 競合だけに掲載。スポカフェへの取り込みを狙う
  5. スポーツバー◎ / ダーツバー◎ / 観戦設備あり（未掲載） … 未掲載だが業態・設備が好相性の新規開拓
  6. 評価◯★（未掲載） … 未掲載で口コミ評価が高い

③ 選んだ店をどう「回る順番」にするか

行く店を選んだ後、機械が地理的に一筆書きに近い順番へ並べ替えます。2段階のアルゴリズムを使います。

1. 最近傍法（Nearest Neighbor）で初期順を作る

起点から「いちばん近い未訪問店」へ、さらにそこから「次に近い店」へ…と貪欲に繋いで、ひとまずの巡回順を作ります。

2. 2-opt で交差をほどく

初期順は所々で経路が交差（＝行ったり来たり）します。2-optは経路の一部区間を反転して総距離が縮むなら採用、を繰り返し、交差を解消します。

最近傍だけ: 起点 ─ A ╲ ╱ C 2-opt後: 起点 ─ A ─ B ─ C ─ D ╲ ╳ ╱ （交差なし・総距離が短い） B D ← 交差して戻っている

この並べ替えにより、実測で歩行距離が大きく短縮されます（テストケースで約6割減）。 「起点に戻る」を指定した場合は、最後に起点へ戻る閉ループとして最適化します。

💡 Googleマップ側でも経路は微調整されますが、訪問する順番そのものをこのツールが最適化して渡すため、 マップ任せより行ったり来たりが減ります。

④ 「すでに行く場所が決まっている」場合

アポ・確定訪問先を指定すると、その地点を中心に候補を探します。アポに時刻を入れると、ルートを次のように組みます。

  • アポ前：開始時刻からアポ時刻までに無理なく寄れる店を、移動時間の余裕（バッファ）を見て配置
  • アポ：固定ポイントとして時間を確保（📌 で表示）
  • アポ後：アポ終了後から終了時刻まで、残りの店を最適順で配置

時刻を入れない場合は、アポ地点を「必ず立ち寄る経由地」として全体の周遊順に組み込みます。

⑤ 時間・距離の見積もり方

  • 移動時間：店間の直線距離を歩行速度で割って算出（設定で調整可）
  • 滞在時間：1店あたり一定時間を確保
  • 指定の活動時間（開始〜終了）に収まる範囲で店を配置します
⚠ 距離は直線ベースの概算です。実際の所要時間は信号・経路・店内滞在で前後します。 最終的な移動は発行されるGoogleマップのルートでご確認ください。
