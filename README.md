# 訪問店舗提案サービス

アポイント（固定点）の前後の空き時間に、近隣の**営業ターゲット店舗**を
移動効率の良いルートで回るプランを自動提案するための社内ツール。

> 例：「西日暮里に19:00アポ。17:00〜22:00の枠で、ついでに回るべき店とルートを提案して」

---

## 実行手順（初回セットアップ）

```powershell
cd "G:\マイドライブ\作業フォルダ2025～\Claude作業フォルダ\Claudecode スポカフェ\訪問店舗提案サービス"

# フェーズ0前半（統合・名寄せ・突合） ※済み
python _scripts/統合マスタ構築.py

# フェーズ0後半（ジオコーディング。中断しても再開可） ※進行中 or 済み
python _scripts/ジオコーディング.py

# フェーズ0.5（品質強化・スコアリング）← ジオコーディング完了後に実行
python _scripts/フェーズ05_品質強化.py
```

## 訪問プラン生成（日常利用）

```powershell
# 対話モード（推奨）
python _scripts/ルートプラン生成.py

# 引数指定モード
python _scripts/ルートプラン生成.py ^
  --appt_name "○○バー" ^
  --appt_addr "東京都荒川区西日暮里5-9-5" ^
  --appt_start 19:00 --appt_end 20:00 ^
  --window_start 17:00 --window_end 22:00 ^
  --owner 鈴村 --max 5
```

出力: `_output/route_plans/訪問プラン_〇〇_日時.html`
→ ブラウザで開き、営業メモを入力して印刷（Ctrl+P）

## 設定変更

`_config/設定.json` を編集してパラメータを調整できます。

| 項目 | 設定キー |
|---|---|
| チームメンバー一覧 | `team.members` |
| 検索半径(m) | `route.search_radius_m` |
| 1店舗あたり滞在時間(分) | `route.stay_minutes_per_store` |
| アポ前バッファ(分) | `route.buffer_before_appt_min` |
| スコア重みづけ | `scoring.base` / `scoring.penalty` |
| S/A/B/C閾値 | `scoring.thresholds` |
